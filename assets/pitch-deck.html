HOW IT WORKS

From JD to shortlist in under 10 minutes

1

JD Submitted

Recruiter pastes a job description. Orchestrator parses it — extracts skills, experience, location, domain — and activates the full swarm.

2

Multi-Source Sourcing

Sourcer Agent searches LinkedIn, GitHub, Naukri, and internal databases simultaneously. Deduplicates candidate profiles across sources. Returns a confidence-ranked list.

3

Semantic Matching

Matcher Agent embeds JD + each profile using BGE-M3 multilingual embeddings. Scores each candidate across 5 dimensions with individual breakdowns and natural-language explanations.

4

Voice Screening

Screener Agent calls top candidates. Natural conversation in Hinglish — adaptive, contextual, fair. Evaluates communication, confidence, and cultural fit. Produces structured assessment report.

5

Auto-Scheduling

Scheduler Agent checks recruiter's calendar, proposes available slots, negotiates with candidates, sends calendar invites and reminders. No email tennis required.

6

Human Review

Recruiter reviews the final shortlist with scoring breakdown, voice assessment report, and call transcripts. One-click feedback to improve future matching.

Communication protocol: All agents communicate via async message bus with standard envelope format (sender, recipient, type, payload, timestamp, message ID). Every message logged. Every decision auditable. Supports pause, resume, and rollback at any pipeline stage.

Key design decisions:

Async messaging — agents never block on each other

State management — orchestrator maintains pipeline state per job (pause, resume, rollback)
